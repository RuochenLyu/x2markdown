# 排障说明

## 右键菜单里没有出现“复制为 Markdown”

可能原因：

- 插件没有成功加载
- 当前页面 URL 不匹配受支持格式
- service worker 没有成功注册或右键菜单没有创建

排查方法：

1. 打开 `chrome://extensions`，确认 `X2Markdown` 已启用。
2. 确认当前页面是以下两种格式之一：
   - `https://x.com/<user>/status/<id>`
   - `https://x.com/<user>/article/<id>`
3. 在扩展页点击一次“重新加载”，再刷新当前页面重试。
4. 在扩展详情页检查 service worker 是否正常运行。

## 点击后提示“未找到正文内容”

可能原因：

- 当前内容不是标准帖子正文区域
- 页面只渲染了壳，没有渲染出正文
- 命中了 quoted post、卡片或其他非正文区域

处理建议：

1. 确认当前页面是支持的详情页，而不是时间线摘要卡片。
2. 滚动页面，等待正文完全渲染后重试。
3. 如果是 `article` 页面，确认标题和正文已经可见。

## 点击后提示“复制失败，请手动重试”

可能原因：

- 浏览器阻止了当前上下文的剪贴板写入
- 页面焦点异常
- 回退方案 `execCommand('copy')` 也没有成功

处理建议：

1. 关闭页面上的其他弹窗后重试。
2. 在页面空白处重新右键，再次执行“复制为 Markdown”。
3. 检查浏览器是否有剪贴板相关限制或安全插件干扰。

## 图片链接不完整或缺失

可能原因：

- 当前内容没有图片
- 图片不是标准 `pbs.twimg.com/media` 附件
- X 改变了图片附件 DOM 结构

处理建议：

1. 确认原帖子正文确实有静态图片。
2. 优先在受支持的详情页测试，而不是时间线摘要卡片。
3. 如果是视频、GIF 或卡片预览，当前版本不会导出。

## 长文页面复制结果混入无关内容

可能原因：

- X 长文阅读视图结构发生变化
- 某些元信息节点被误判成正文块

处理建议：

1. 先确认是否是受支持的长文页面，包含 `x.com/<user>/article/<id>` 或长文阅读视图形式的 `status` 页面。
2. 检查是否混入了统计数字、作者区或推荐内容。
3. 如果稳定复现，优先在该页面结构上调整 `content.js` 的正文块过滤规则。
